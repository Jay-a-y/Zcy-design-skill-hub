## Cross-cutting observations

- A persistent top bar and left navigation establish the application frame.
- Main content uses a strong title, short description, large section gaps, and bordered demo areas.
- A narrow right rail anchors long pages without competing with content.
- Blue consistently marks links, current navigation, selected state, progress, and primary actions.
- Most content surfaces remain white; pale gray fills separate examples, code, and inactive regions.
- Borders are thin and shadows are rare outside overlays.
- Component documentation demonstrates state breadth, not merely the happy path.
- Tables and API references favor compact rows, stable columns, and understated borders.
- Data-entry controls share a small/default/large size language and consistent blue focus/selected states.
- Read-only, disabled, loading, controlled, cleared, error, and custom-render variants are treated as first-class states.
- Complex values preserve context through prefixes, suffixes, units, paths, tags, marks, previews, and formatted display.
- Forms combine compact controls with generous vertical separation between conceptual sections.
- Display components favor structured alignment and restrained borders over decorative containers.
- Component choice follows the reading task: comparison, scanning, hierarchy, detail, chronology, or contextual explanation.
- Loading, empty, error, overflow, preview, expansion, selection, and long-data behavior are treated as part of the default design.
- Rich media and colored status elements remain secondary to text and data meaning.
- Feedback components form a disruption hierarchy from inline state through blocking overlays.
- Loading treatment mirrors the scope and known structure of the pending content.
- Global configuration and business wrappers standardize behavior across modules rather than restyling each page.

## Adding a batch

Use filenames `NN-subject.png` continuing the sequence. Add one row per image and add only genuinely new cross-cutting observations. If the new batch introduces forms, data display, feedback, overlays, or charts, create a focused component reference and link it directly from `SKILL.md`.
