# 项目插图与空状态兜底规则

本文件规定云随 Demo 在缺少正式 Figma 插图时，如何保持页面完整性，同时避免 AI 自行创造不稳定视觉。

## 核心原则

1. Demo 完整性优先，但不伪装成正式项目插图。
2. 缺少插图本身不阻断 Demo 生成；只有插图承载核心业务含义、流程判断或风险决策时，才输出设计缺口。
3. 正式 Figma 或项目品牌资产存在时，必须优先使用正式资产。
4. 无正式资产时，使用 Skill 内置的 UDesign Empty 通用状态插图或云随中性占位。
5. 禁止 AI 自行生成机器人脸、笑脸、徽章、随机插画或装饰图。

## 来源优先级

1. 正式 Figma 中的项目插图或页面视觉。
2. 云随项目已沉淀的正式插图资产。
3. UDesign Empty 官方状态 SVG。
4. 云随中性占位结构。
5. 设计缺口。

## 无 Figma 时的默认展示

| 页面/状态 | 默认处理 |
| --- | --- |
| 暂无数据、暂无记录、未开始 | 使用 `empty.svg` |
| 成功、已完成、处理完成 | 使用 `success.svg` |
| 失败、超时、执行失败 | 使用 `fail.svg` |
| 网络异常、系统连接异常 | 使用 `network-error.svg` |
| 维护中、当前能力暂不可用 | 使用 `maintenance.svg` |
| Loading、处理中、排队中 | 不用大插图；使用 `Steps + Progress + Loading + 业务文案` |
| 业务主视觉页、首页能力宣传区 | 使用云随中性占位，并标记“项目插图待补充” |
| 结果详情页 | 优先展示结构化结果、表格、依据和原文定位；不强行增加插图 |

## 云随中性占位

当 UDesign Empty 状态图也不适配，但页面仍需要视觉占位时，使用低存在感结构：

- 圆形或方形浅色底。
- 内部放置 UDesign 官方图标。
- 颜色使用 UDesign Token。
- 不添加拟物、卡通、人物、机器人或发光装饰。
- 下方必须有明确标题、说明和下一步操作。

示例语义：

```text
图形：浅蓝圆底 + 官方 file-search 图标
标题：采购文件智能检测
说明：说明当前状态、材料要求、可执行操作
操作：开始检测 / 返回 / 查看历史记录
备注：项目插图待正式设计补充
```

## 何时输出设计缺口

只有以下情况才因插图或空状态输出设计缺口：

- 插图承担业务解释，例如风险等级、证据链、审核结论的核心表达。
- 插图是页面品牌主视觉，且会影响首页或核心入口的识别。
- 需求明确要求还原某个 Figma 插图，但 Skill 未收录资产。
- 产品或研发要求“正式视觉稿级别”交付，而非 Demo 验证。

普通空状态、失败态、维护态缺少 Figma 插图时，不输出设计缺口，直接使用 UDesign Empty 兜底并记录说明。

## 单 HTML 实现要求

生成单 HTML Demo 时：

1. 只内联当前页面实际使用的状态 SVG。
2. 不将全部插图资产塞入 HTML。
3. 保留 SVG `viewBox`，允许用 CSS 控制尺寸。
4. 默认插图尺寸：空状态 96px；页面主状态 120px；不要超过 160px，除非 Figma 明确如此。
5. Loading/处理中优先显示进度与步骤，不用插图占位。
