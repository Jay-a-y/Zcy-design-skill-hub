# 页面骨架与可复用模式

## 全局宿主骨架

云随是业务系统内从右侧展开的 1200px 工作台，而不是脱离宿主的普通全屏站点。正式样板的稳定结构：

- 顶部 64px 蓝色渐变品牌栏：云随标识、项目名称/采购方式、组织、账户管理、关闭。
- 二级页面再使用 60px 白色页头：返回、服务名称、右侧 AI 免责声明。
- 主内容区使用浅灰背景、24px 外边距和白色内容容器。
- 整体框架映射 UDesign `Drawer + Layout + Header + Content`；基础交互使用 UDesign，布局以正式 Figma 为准。
- 主色使用 `--ud-brand-color`（默认 `#0052D9`）；品牌栏局部使用 `#3154E5 → #4769F3` 渐变。
- 正文以苹方为主；不要把生成代码里的临时样式当作新 Token。

## P01 首页：阶段 + 服务卡片

适用：一个项目阶段下展示 2 项、5 项或未来更多服务。

稳定结构：项目阶段名称和说明、服务卡片网格、卡片能力摘要、可用/不可用状态。服务增多时卡片换行扩展，不改品牌栏和阶段信息结构。未来扩展示意中的重复卡片只是布局占位，不是新增服务。

## P02 服务起始页：引擎就绪

适用：采购文件检测、围串标审查、辅助评标等可主动发起的 AI 服务。

稳定结构：状态插画/品牌视觉、结果导向标题、一项明确主操作、三张能力说明卡。能力卡用于解释任务价值，不能替代发起前的条件与风险说明。

## P03 当前阶段不可用

适用：服务存在，但当前项目阶段不允许使用。

稳定结构：锁定/不可用插画、明确服务开放阶段、当前不可用原因、返回首页。不要与无权限、失败、无历史记录混用。

## P04 外部触发等待

适用：云随依赖业务系统先发起请求。

稳定结构：说明尚未收到请求、给出业务系统里的具体发起动作、同时提供返回首页和返回业务系统。不要显示一个无效的云随“开始”按钮。

## P05 单文件上传与一次性提取

适用：采购文件提取。

稳定结构：上传区域、格式/大小限制、一次性结果提示、返回。上传失败时进入可恢复失败模式；处理成功后可进入文档预览 + 字段提取结果。

## P06 多材料上传检测工作台

适用：需要多个文件、文件夹或压缩包，并在检测前处理材料和匹配场景。

稳定结构：

1. 顶部项目名与“开始检测”。
2. 上传区：文件/文件夹、压缩包、格式和总容量说明。
3. 左侧上传队列：文件、大小、状态和单文件进度。
4. 右侧材料处理管线：读取、整理、提取、归类、查询主体、等待确认。
5. 智能匹配场景和“手动选择审查项”。
6. 抽屉承载审查项多选，不把大量选项直接堆进主页面。

## P07 长任务进度

长任务不使用单一百分比解决所有服务。

- 文件提取：左侧原文预览，右侧展示解析、章节识别、字段提取等子步骤。
- 文件检测：项目数据 + 总体进度/耗时/完成数量 + 阶段管线 + 当前检测项。
- 围串标与辅助评标：沿用“总进度 + 阶段 + 当前任务”的信息层级，但具体步骤必须来自正式样板或需求，不自行编造。

## P08 并发排队

适用：服务并发已满。显示正在排队、当前队列位置、保持页面且不要重复提交，并提供“取消排队并返回首页”。不要伪装成普通 Loading。

## P09 失败与超时

- 上传/文件处理失败：指出失败对象，提供重新上传或针对性重试。
- 任务超时：明确本次已失败，需要重新发起；不能继续显示“处理中”。
- 失败页都保留返回路径，且不得丢失仍可复用的用户输入，除非技术约束明确说明。

## P10 结果与详情

结果详情页使用云随骨架，并至少包括摘要、分组/筛选、依据、来源、原文或字段定位和后续动作。优先使用 `Descriptions + Tabs + Table/List + Collapse + Drawer + Anchor`；无法映射的结果交互使用 quick-use 兜底。线上 AI 截图只能参考内容框架。

## P11 阶段已过后的历史态

- 有记录：使用提示条说明当前展示历史回填/执行记录；结果全只读；禁止重新发起和一键回填。
- 无记录：显示“暂无提取记录/暂无检测记录”等具体名称，说明该项目在有效阶段没有使用服务，仅保留返回首页。

## P12 宿主系统联动

- 在采购文件管理详情中可展示云随免费使用提示。
- 在宿主表单字段旁以“AI已回填”标记来源。
- 宿主提示和回填标签必须沿用宿主系统组件，不套用云随工作台卡片样式。

## 模式组合规则

- 成熟单一模式：直接复用一个模式，替换服务、阶段和真实业务内容。
- 成熟组合模式：可组合 P04/P06/P07/P10 等模式，但先梳理状态流并确认跨页返回关系。
- 未覆盖的复杂核心模式：如果需要新增首页骨架、全新结果交互或新的高风险决策模式，输出设计缺口，等待设计师补充正式样板，不能从多个旧页面拼凑成事实标准。
