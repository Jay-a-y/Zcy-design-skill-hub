# 反馈说明

本说明用于记录云随 Skill 试用反馈。

## 反馈字段

```markdown
# 云随 Skill 试用反馈

- 试用人：
- 日期：
- 使用环境和模型：
- 需求场景：
- 是否成功生成：
- 首次生成耗时：
- 是否存在 P0 问题：
- 关键状态是否覆盖：
- 人工修改轮次：
- 人工介入时间：
- 最大问题：
- 是否需要设计师介入：
- 建议补充内容：
- 问题初步归因：
```

## 初步归因选项

- 需求输入不足。
- Skill 知识缺失。
- Figma 样板缺失。
- UDesign 映射缺失。
- AI 执行错误。
- 单次特殊需求。
- 可沉淀的重复稳定模式。

## 处理建议

- 单次内容错误：优先修改本次 Demo。
- 输入不足：补充需求或测试数据。
- AI 执行错误：保留原始输入和首次输出，复测是否稳定复现。
- 样板或核心规则缺失：反馈设计师补充 Figma 或 Skill。
- 重复稳定模式：设计师评估是否回灌到新版本。

不要制造虚构测试结果。没有真实数据的字段保持空白或写“未记录”。
