# 业务试用说明

本说明用于第一轮业务试用。

## 建议试用范围

- 建议试用人数：1 至 2 名云随产品或研发。
- 首轮场景：成熟页面、成熟组合流程。
- 不要求第一轮覆盖所有云随场景。

## 试用目标

验证别人是否能独立完成：

- 下载 Skill。
- 安装或放置 Skill。
- 调用 Skill。
- 输入需求和测试数据。
- 生成单 HTML Demo。
- 判断 P0/P1/P2 和设计缺口。

## 建议场景

- 成熟页面：采购文件提取上传或阶段不可用。
- 成熟组合流程：通用上传、材料处理、审查项抽屉、检测进度、结果定位。
- 异常状态：上传失败、超时失败、并发等待、历史无记录。
- 宿主系统：免费使用提示、AI 已回填标签。

## 试用记录

每次试用保留：

- 原始输入。
- 首次生成结果。
- 修改轮次。
- 最终结论。
- 是否需要设计师介入。

单次试用失败不直接归因于 Skill。先判断是需求输入不足、AI 执行错误、样板缺失、映射缺失还是 Skill 知识缺失。
