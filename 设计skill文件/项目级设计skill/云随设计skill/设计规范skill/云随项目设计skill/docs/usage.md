# 日常使用说明

本说明面向云随产品经理和研发。

## 1. 准备材料

准备：

- 需求文档、业务诉求或原型说明。
- 真实业务文件或测试数据。
- 必须覆盖的页面或流程。
- 是否要求单 HTML。
- 特别限制，例如不要创建工程脚手架、必须覆盖异常状态等。

不需要自行判断 L0～L3，不需要重新整理 Skill 已收录的 Figma 节点，也不需要指定 UDesign 组件。

## 2. 加载云随 Skill

在 AI 工具中加载或指向云随项目级设计 Skill。若平台支持 Skill 语法，使用 `$yunsui-design`。

## 3. 复制生成提示词

复制 [generate-demo.md](../ai-prompts/generate-demo.md) 中的模板，并填入本次需求。

## 4. 生成单 HTML Demo

默认要求 AI 输出一份可离线运行的单 HTML Demo。只有明确需要工程化实现时，才让 AI 改造或生成代码工程。

图标已内置在 Skill 的 `assets/icons/tdesign-icons/` 中。生成 Demo 时要求 AI 只使用本地 UDesign 官方 SVG，并只内联本次实际用到的图标，不要使用 emoji、第三方图标库或 AI 自行手写 SVG。

普通空状态、成功、失败、网络异常和维护中页面也已内置 UDesign Empty 官方状态 SVG。没有 Figma 插图时，可让 AI 先用这些通用状态图或云随中性占位完成 Demo；只有插图影响核心业务表达或强品牌主视觉时，再反馈设计师补充。

## 5. 查看结论

重点查看：

- 已覆盖页面和状态。
- P0/P1/P2 结论。
- 关键假设。
- 产品待确认项。
- 设计缺口。
- 云随内部补全范围。
- 图标是否来自本地 UDesign 官方 SVG。
- 空状态/插图是否使用正式 Figma、UDesign Empty 或云随中性占位。

## 6. 修改普通问题

如果只是文案、字段、测试数据或普通状态遗漏，可以直接要求 AI 修改 Demo。

## 7. 回流复杂模式

出现以下情况时反馈设计师：

- 新增复杂核心页面。
- 新增角色、阶段或权限规则。
- 新增风险判断机制或关键结果交互。
- 正式 Figma 样板缺失。
- 同类问题在多个需求中重复出现。

反馈模板见 [feedback-guide.md](feedback-guide.md)。
