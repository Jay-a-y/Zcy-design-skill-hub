# 输出规则

本文件规定云随 Skill 的默认交付内容。

## 默认输出

日常生成或改造完成后输出：

- 可运行单 HTML Demo 或修改后的文件。
- 已覆盖页面和状态。
- 关键假设。
- 产品待确认项。
- 云随内部补全范围。
- 设计缺口。
- P0/P1/P2 结论。

没有设计缺口时不强行输出空报告。

## 单 HTML Demo

用户说“生成 Demo”或“生成 HTML”时，默认交付一份可离线运行的单 HTML：

- 内联 CSS、JS、图标和必要轻量素材。
- 图标必须来自 [icon-library.md](../knowledge/icon-library.md) 指向的本地 UDesign 官方 SVG；只内联本次实际用到的图标。
- 空状态或状态插图按 [illustration-rules.md](../knowledge/illustration-rules.md) 执行；普通缺图不阻断 Demo 生成。
- 双击即可打开。
- 不需要安装依赖。
- 不需要启动服务。
- 不创建 React、Vite、Next.js、Vinext 或其他前端工程。

单 HTML Demo 用于产品验证、研发沟通和设计评审，不等同于生产代码。

## 数据与假设

- 测试数据优先使用用户提供的数据。
- 不输出虚构业务数据作为事实。
- 如需模拟数据，必须明确标注为模拟数据。
- 关键业务字段、结果结论和风险依据不能凭空编造。

## 生成前后的表达

- 不在生成前输出冗长分析代替实际交付。
- 可以简要说明覆盖判断和关键假设。
- 交付后说明文件位置、覆盖状态、缺口和 P0/P1/P2 结论。

## 设计缺口

出现未覆盖或全新复杂模式时，按 [design-gap-output.md](../ai-prompts/design-gap-output.md) 输出设计缺口。成熟部分可以生成，但不得把临时方案当成正式云随规则。

## 演示控制台

若 Demo 需要身份、项目阶段、页面状态或快捷入口切换，必须做成外置演示控制台。控制台可底部悬浮、可隐藏，但不得进入云随业务页面内容区，也不得挤占正式页面布局。

## 图标资产

- 不输出 emoji、AI 手写 SVG 或第三方图标库作为云随正式图标。
- 不把全量图标库写入单 HTML。
- 若缺少可准确表达业务语义的官方图标，输出“图标待补充”或使用近义替代并说明。

## 插图与空状态

- 正式 Figma 插图优先。
- 无正式插图时，通用空状态、成功、失败、网络异常和维护中使用 Skill 内置 UDesign Empty 状态 SVG。
- Loading、排队和处理中优先使用步骤、进度和业务文案，不用 AI 自造插画。
- 使用 UDesign Empty 或云随中性占位时，在交付说明中简要记录兜底范围。
- 只有插图承载核心业务含义、风险解释或强品牌主视觉时，才作为设计缺口阻断生成。
