# 评审 Demo 提示词模板

适用对象：产品经理、研发或设计师。

```text
请使用 $yunsui-design 评审这份云随 Demo，不要先改动文件。

## 待评审 Demo
[粘贴文件路径、HTML 内容、截图说明或可访问地址]

## 对应需求
[粘贴需求文档或业务诉求]

请检查：

1. 业务对象和信息架构是否清楚。
2. 用户角色、项目阶段、权限和可执行操作是否正确。
3. 开始、等待、排队、Loading、结果、失败、超时、不可用、无权限和历史态是否按需覆盖。
4. 已有页面是否优先复用正式 Figma 样板和云随成熟页面模式。
5. 基础组件和交互是否符合 UDesign 映射。
6. 图标是否来自 Skill 内置 UDesign 官方 SVG，是否存在 emoji、第三方图标库或 AI 自行手写 SVG。
7. 空状态和项目插图是否按正式 Figma 优先、UDesign Empty 兜底、云随中性占位的顺序处理，是否存在 AI 自造卡通插画。
8. 云随内部补全策略是否只用于非核心缺失部分，是否存在越界。
9. 演示用身份、阶段或状态切换器是否外置，是否误入业务页面内容区。

请输出：

- 结论：通过 / 修改后通过 / 不通过。
- P0 阻断问题。
- P1 重要问题。
- P2 优化问题。
- 修改建议。
- 是否存在设计缺口。
- 是否建议回灌 Skill。

普通问题给出修改建议，不默认由设计师代做；只有缺少核心样板、业务规则或稳定模式时才列为设计缺口。
```
