# 生成 Demo 提示词模板

适用对象：产品经理或研发。

```text
请使用 $yunsui-design 生成云随 Demo。

## 需求文档
[粘贴需求文档，或说明文档路径]

## 业务诉求
[说明本次要解决的问题、业务目标和期望验证点]

## 用户角色
[采购单位 / 代理机构 / 专家 / 未明确]

## 项目阶段
[需求确定 / 文件编制 / 开标流程 / 评标 / 履约 / 未明确]

## 测试数据
[粘贴字段、样例数据、业务文件路径或说明无真实数据]

## 期望页面或流程
[说明要生成单页面、完整流程、已有页面改造或多状态演示]

## 是否要求单 HTML
[是 / 否；默认是]

## 特别限制
[例如不要创建工程脚手架、必须覆盖失败态、不要使用模拟数据等]

请自行读取 Skill 中的知识文件和执行指南，自动判断覆盖范围。已覆盖时生成可运行 Demo；部分覆盖时只允许非核心内容使用云随内部补全策略并说明范围；未覆盖或全新复杂核心模式时输出设计缺口，不要自行创造正式方案。

图标必须来自 Skill 内置的 UDesign 官方 SVG 图标库，只内联本次实际使用的 SVG；不要使用 emoji、第三方图标库或 AI 自行手写 SVG。

如果没有正式 Figma 插图，普通空状态、成功、失败、网络异常和维护中页面请使用 Skill 内置的 UDesign Empty 状态 SVG 或云随中性占位保持 Demo 完整性；不要因为普通缺图中断生成。只有插图承载核心业务含义或强品牌主视觉时，才输出设计缺口。

不要要求我判断 L0～L3，不要要求我重新整理 Skill 已收录的 Figma 节点，也不要把产品 Demo 当作正式视觉依据。
```
